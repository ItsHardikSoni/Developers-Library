Chapter.5 - Scope Rules of Variables

Scope Rules of Variables

Scope Rules – The range of the code in a program over which a variable has meaning is called the scope of the variable. In simple terms, the scope of a variable determines how it can be accessed by different parts of the program.

There are three types of scope normally in C:

  • Local Scope
  • Function Scope
  • File Scope and Global Scope

Types of Scope Rules

Here we define all the types of Scope Rules:

Local Scope

A block of statements in C is surrounded by curly braces, i.e., { and }. We can declare variables inside the block; such variables, called local variables, can be referenced only within the body of the block.

When the control reaches the open braces, these variables come into existence and get destroyed as soon as the control leaves the block through the closing brace.

So the local variables are available to all the blocks enclosed by the block in which they have been declared.

        while (flag) {
            int x;
            if (i > x) {
                int j;
                // -
            }
        }
  • The scope of x is the while block, and j is the if block.
  • The variable x can also be referenced in the if block because this block is completely enclosed in the while block.
  • On the other hand, the scope of j is only the if block. Since a function is a block in itself, all the variables declared in it have local scope.
  • The formal parameters in a function have local scope.

Function Scope

  • The function scope pertains to the labels declared in a function in the sense that a label can be used anywhere in the function in which it is declared.
  • Thus, we can use different labels with the same names in different functions.

File Scope or Global Scope

Since the global variables are declared outside all blocks and functions, they are available to all the blocks and functions.

Thus, the scope of a global variable is the entire program file. This type of scope is known as file scope.

e.g. global variables rate and qty in the following have file scope:

        int qty, rate;
        void main() {
            int gross, net;
            // -
        }
        
        void fun() {
            char c;
            // -
            { 
                int a;
                // -
            }
        }

Scope Rules Summary

  • A local variable can be accessed only within the block in which it is declared. The local variable declared in a function exists only during the execution of the function. Therefore, these variables are also known as automatic because they are automatically created and destroyed.
  • The scope of formal parameters is local to the function in which they are defined.
  • Since the global variables are not declared in a particular function, they are available to all the functions. Thus, the scope of a global variable is the entire program.
